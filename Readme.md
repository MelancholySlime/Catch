# Catch

A simple utility to convert and extract color palettes from HackSaw into Quartz-compatible format.

---

## ✨ Overview

**Catch** is a lightweight tool designed to:

* Convert palette data from HackSaw's legacy database format
* Transform it into Quartz's modern palette schema
* Split multiple palettes into individual JSON files

This enables seamless reuse of palettes between two different tools.

## 📂 File Locations

### HackSaw (Input)

```plaintext
C:\Users\<YourUser>\AppData\Roaming\hacksaw\SampleDB.json
```

This file is automatically generated by HackSaw and contains:

* Multiple palettes
* Stored in a legacy structure

---

### Quartz (Output)

```plaintext
C:\Users\<YourUser>\AppData\Roaming\Quartz\palette\
```

Quartz expects:

* One JSON file per palette
* New schema format

---

## ⚙️ How It Works

### 1. Input Structure (HackSaw)

* Stored as an array of palette objects
* Each palette includes:

  * `Name`
  * `Palette[]` (list of color entries)

Each color entry contains:

* `r, g, b` (0–255)
* `a` (0–100)
* `vec4` (normalized values 0–1)
* `time`

---

### 2. Transformation Logic

Each palette is converted into Quartz format using the following mapping:

| HackSaw   | Quartz                  |
| --------- | ----------------------- |
| Name      | name                    |
| Palette[] | colors[]                |
| vec4[0-3] | r, g, b, a              |
| time      | time                    |
| (none)    | hex                     |
| (none)    | mode = "random"         |
| (none)    | created = ISO timestamp |

---

### 3. Important Notes

* `vec4` is used instead of raw RGB to preserve normalized color accuracy
* `hex` is generated from RGB values (converted from vec4)
* Alpha (`a`) is taken directly from vec4
* Missing values are handled safely with defaults

---

### 4. Output Behavior

* Each palette is exported as a **separate JSON file**
* Each file contains exactly **one palette**
* Filenames are:

  * Based on palette name
  * Sanitized to remove invalid characters
  * Appended with an index for uniqueness

## 📦 Example

### Input (HackSaw)

```json
{
  "Name": "Example",
  "Palette": [
    {
      "vec4": [1, 0, 0, 1],
      "time": 0
    }
  ]
}
```

---

### Output (Quartz)

```json
{
  "name": "Example",
  "mode": "random",
  "created": "2026-04-19T00:00:00.000Z",
  "colors": [
    {
      "r": 1,
      "g": 0,
      "b": 0,
      "a": 1,
      "time": 0,
      "hex": "#ff0000"
    }
  ]
}
```

## 🚀 Usage

### 1. Copy Source File

Copy your HackSaw database file:

```plaintext
C:\Users\<YourUser>\AppData\Roaming\hacksaw\SampleDB.json
```

into the project folder.

---

### 2. Run Script

```bash
node convert.js
```

---

### 3. Get Output

* Generated files will appear in the `/output` folder

---

### 4. Import into Quartz

Copy all generated `.json` files into:

```plaintext
C:\Users\<YourUser>\AppData\Roaming\Quartz\palette\
```

---

## ⚠️ Edge Cases Handled

* Missing or null values
* Empty palettes
* Invalid filename characters
* Inconsistent alpha values
* Unexpected data structure issues

---

## 🧠 Why This Exists

HackSaw and Quartz use incompatible palette formats.

**Catch** bridges that gap by:

* Converting schema automatically
* Normalizing color data
* Splitting palettes into usable files
